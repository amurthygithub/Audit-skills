## Cross-Referencing System

Every workpaper must cross-reference: source documents, lead schedules, risk-to-response linkage, tickmark-to-workpaper, conclusion linkage. Format: [Source WP] -> [This WP] -> [Conclusion WP].

## Tickmark System (8 Standard)

| Symbol | Meaning | Documentation |
|--------|---------|---------------|
| check | Verified to source document | Identify source |
| wedge | Agrees to prior year | Identify prior-year WP |
| down-arrow | Footed / cross-footed | Specify computation |
| circle | Tested via sampling | Reference sampling WP |
| square | Confirmed with third party | Identify confirment |
| triangle | Recomputed / recalculated | Show computation |
| reference | Exception noted | Describe exception |
| diamond | Re-performed | Describe procedure |

Custom tickmarks permitted but must be defined in legend (typically B-5).

## Sign-off Protocol (AS 1215.06)

| Role | Sign-off Required | Reference | Timing |
|------|-------------------|-----------|--------|
| Preparer | Initials + date performed | AS 1215.06 | At time of work |
| Reviewer | Initials + date reviewed | AS 1201 | Before report release |
| Engagement Partner | Sign-off on significant matters | AS 1215.15 | Before report release |
| EQR Partner | Sign-off per AS 1220 | AS 1220 | Before report release |

## Three Mandatory Elements (AS 1215.04)

1. Clear PURPOSE — Why was this workpaper created?
2. Clear SOURCE — Where did the data come from?
3. Clear CONCLUSIONS — What was the auditor's conclusion?

## Experienced Auditor Standard (AS 1215.06A)

Documentation must enable an experienced auditor (no prior engagement connection) to understand: nature/timing/extent of procedures, evidence obtained, conclusions, who performed/reviewed and when.

## 5W1H Framework

| Question | Content | Standard |
|----------|---------|----------|
| Who | Preparer, reviewer | AS 1215.06 |
| What | Procedures, evidence, conclusions | AS 1215.06 |
| When | Date performed, date reviewed | AS 1215.06, .15 |
| Where | Source of data, location of records | AS 1215.10 |
| Why | Purpose, assertion addressed | AS 1215.04, .07 |
| How | Methodology, sampling approach | AS 1215.04, .10 |

## Header/Footer Requirements

Every page: client name, engagement period, WP index, preparer initials/date, reviewer initials/date, Page X of Y.
