# Industry: SaaS / Technology (pre-IPO, SOX 404 readiness)

## Posture
Pre-IPO SaaS companies preparing for SOX 404 compliance and eventual PCAOB oversight. Building workpaper infrastructure from scratch. Often transitioning from review-only to audit engagements. Revenue recognition (ASC 606) complexity.

## Workpaper Infrastructure Priorities
1. Establish permanent file (A-series): entity information, org structure, capitalization table, related parties.
2. Design sampling methodology for recurring audit procedures (AR, revenue, deferred revenue, payroll).
3. Build internal control documentation (C-series): COSO 5 components, ITGCs for cloud-native infrastructure.
4. Create electronic workpaper platform with version control, access controls, and audit trail.
5. Create IPE validation procedures for SaaS-specific data (subscription billing systems, usage-based revenue).

## Typical Sampling
MUS for AR (contract receivables). Variables sampling for deferred revenue testing. Attribute sampling for controls over billing system. Full-population testing via data analytics where feasible.

## Pain Points
- No prior-year workpapers to reference
- Revenue recognition complexity (ASC 606) drives larger samples
- ITGCs for cloud-native infrastructure (CI/CD, infrastructure-as-code, container orchestration)
- IPE validation for usage-based billing data
- Building tickmark legends and cross-reference indices from scratch

## Top Use Cases
- UC-01: MUS sampling workpaper for AR
- UC-02: 5-part finding for control gap
