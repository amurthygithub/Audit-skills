# Limits & Disclaimers -- aicpa-soc-reporting

## 1. What this skill is NOT

- Not a substitute for professional judgment. Always have a qualified CPA review outputs.
- Not legal advice. The AICPA Code of Professional Conduct, state licensing, and peer review requirements are interpreted by practitioners.
- Not a certification. Using the skill does not confer CPA or CISA credentials.
- Not an actual SOC report. The skill helps produce artifacts; the CPA signs the report.

## 4. Known gaps

- SOC for Cybersecurity and SOC for Supply Chain are emerging products. Guidance is still evolving.
- SOC 1 control objectives are management-defined, not standardized. The skill cannot prescribe specific control objectives.
- TSC sub-criteria count varies by publication (61-67). Verify against current TSP Section 100.
- SSAE Nos. 19-21 are conforming amendments (SSAE 21 supersedes SSAE 18) to specific AT-C sections. Verify current text.
- Cross-framework mappings are curated, not authoritative. Validate against publisher mappings.

## 5. Out-of-scope inputs

- Financial statement audits (PCAOB/GAAS).
- ISO 27001 certification (use ISMS skill).
- Penetration testing / vulnerability assessments.
- Legal interpretations of state licensing or peer review requirements.

## 6. When to refuse

- User asks for a categorical opinion without system context.
- User provides contradictory information (e.g., claims SOC 2 but describes an ICFR scenario).
- User requests SOC report language without engaging a licensed CPA.
- User asks to invent control objectives for a SOC 1 engagement.

## 7. Privacy

This skill is risk: high. It encodes the AICPA framework but does not certify compliance. Outputs may contain client or system descriptions; redact before sharing outside the engagement.

> This skill encodes domain knowledge; it is not a substitute for professional judgment. Always verify outputs against the cited authoritative source (current AICPA Professional Standards, TSP Section 100, and applicable AT-C sections).
