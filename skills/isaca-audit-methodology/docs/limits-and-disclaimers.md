# Limits & Disclaimers - isaca-audit-methodology

## 1. What this skill is NOT

- **Not a substitute for professional judgment.** Always have a qualified CISA or equivalent review outputs.
- **Not legal advice.** Regulatory compliance certifications require licensed professionals.
- **Not a certification.** Using the skill does not confer CISA, CISM, or any other credential.
- **Not an audit opinion.** The skill produces methodology, templates, and structured outputs; the audit opinion belongs to the practitioner.

## 4. Known gaps

- **No GAGAS/Yellow Book layer**: state/local legislative audit offices need an ISACA-to-GAGAS bridge (independence, CPE, performance-audit reporting); tracked as a structural ticket.
- **No public-sector or healthcare worked example**: UC-01/02/03 cover SaaS and banking only.
- **Ethics principle count** may vary (7 or 8 principles depending on edition).
- **COBIT alignment goals** may not map 1:1 to enterprise goals; consult official COBIT 2019 documentation.
- **Cross-framework severity mapping** (ISACA to PCAOB/COSO) is directional, not exact; severity never auto-derives an ICFR deficiency classification (see chunks/05).
- **Stub executor is canned**: Path 2 returns seed-derived fixtures regardless of evidence text; it demonstrates output structure, not analysis.

## 5. When to refuse

The skill should be used cautiously or not at all when:

- The user asks for a categorical answer without audit context.
- The user provides contradictory information.
- The user asks for an ISACA standard/guideline number the skill cannot verify against the current ITAF publication.
- The user expects a signed audit opinion.

> This skill encodes domain knowledge; it is not a substitute for professional judgment. Always verify outputs against the cited authoritative source.
